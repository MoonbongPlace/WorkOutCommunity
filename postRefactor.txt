## Post 이미지 다중 첨부 리팩토링

### Post.java
- Before: `String image` (단일 URL)
- After: `List<String> images` + `@Convert(StringListConverter)`, 컬럼 길이 2048, `create()` 버그 수정

### StringListConverter.java (신규)
- After: `List<String>` ↔ JSON 직렬화 JPA 컨버터

### CreatePostRequest / UpdatePostRequest
- Before: `String image` 필드 존재
- After: `image` 제거. UpdatePostRequest에 `List<String> keepImages` 추가

### PostService.java
- Before: `MultipartFile profileImage` 단일 파일 처리
- After: `List<MultipartFile> images` 처리, `uploadImages()` 헬퍼로 최대 6장 초과 시 예외 발생

### PostController.java
- Before: create는 `profileImage` 단일 파일, update는 `@RequestBody` JSON
- After: create/update 모두 `multipart/form-data`, `List<MultipartFile> images` 수신

### Result DTOs (Create/Detail/Update/ListItem)
- Before: `String image`
- After: `List<String> images`

### ResponseCode.java
- After: `POST_IMAGE_LIMIT_EXCEEDED` 추가

### DB
- Before: `image VARCHAR(512)`
- After: `image VARCHAR(2048)` (JSON 배열 저장)
